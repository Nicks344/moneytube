После запуска в папке появится файл cli.exe
Основной параметр: -cmd, он обязателен и обязательно должен идти первым. Все остальные аргументы можно располагать как угодно. Параметры указываются так: -{arg}={val}

Есть помощь, она вызывается аргументом -help
Например, чтобы посмотреть какие аргументы можно вводить с командой создания новой задачи на загрузку, нужно ввести:
./cli.exe -cmd=AddUploadTask -help

Для некоторых параметров есть описания на английском, для некоторых - нет, т.к. они кастуются прямо со структуры данных. Для нормального использования конечно лучше их описать. -help - автоматическая команда и информацию берет из метатегов структур данных, так что сам процесс написания описаний не сложный, но ресурсоемкий. Можете сказать какие будут часто использоваться и я их опишу. Порядок расположения аргументов в помощи поменять нельзя, к сожалению, он всегда в алфавитном порядке.
Есть еще параметр -print-tmpl-json, он печатает структуру json, которую нужно подставлять в -tmpl-json. Но работает так себе, для массивов данных пишет null, я решил не заморачиваться и не исправлять это
Есть параметры -tmpl-json и -tmpl-file, они задают json основных данных, шаблон. В -tmpl-json нужно подставлять json в виде текста, в -tmpl-file можно поставить путь к файлу, в котором этот шаблон. В начале берутся данные шаблона, затем на получившуюся структуру применяются параметры командной строки

Итак, чтобы добавить новую задачу на загрузку, нужно ввести:
./cli.exe -cmd=AddUploadTask -tmpl-file=template.json -account-ids=1,2,3,4
